What is Tkinter?

This site is a work in progress.

Well, if you are on this site, you probably know what it is, but you can read about it in detail here.

Tkinter is Python's built-in standar GUI framework, which wraps the tk GUI toolkit. Tk is written in Tcl but also used in other languages (Python, Ruby, Perl). Tkinter is included with standard Windows, Linux and Mac installs of Python, so you don't need to install it separately, and it's cross-platform. It's name, tkinter is comes from the composition of Tk + interface.

A short introduction

Most of the time, the tkinter and the ttk modules are needed when using tkinter, but there are some accessories, such the filedialog and the messagebox.

The tk interface itself is located in a binary module called _tkinter. This module contains the low-level interface to tk, and should never be used directly by application programmers. It is usually a shared library, or DLL, but might in some cases be statically linked with the Python interpreter.

There's a module, tkinter.constants, where all tkinter constants are defined that can be used for keywords, but of course we can also use strings. If you import tkinter this module is automatically imported, so you don't have do it separately.

And now a few words about importing, because many people (rightly) uses it incorrectly. If only tkinter is imported, you can make a well-functioning GUI, but you won't be satisfied with its appearance. It could be compared with Windows 95. It was not today, mainly not in design. Therefore, tkinter (ie tk) has an extension module, called ttk. With that you can make quite modern (platform dependent) looking apps. This appearance is native to the operating system, so it looks different from Windows and Mac. Unfortunately, on Linux there's not a big difference between the plain tkinter and the ttk widgets.

Another good thing that in ttk can not only make native apps, but you able to set different themes (hence its name: themed tk).
Last but not least, ttk also has some new widgets compared to tkinter, but others are missing from it, so pay attention to which module you are using for the widget. So return to imports. most of tkinter applications are starts as: 

from tkinter import *

but a "well-done" tkinter app should begin like this: 

import tkinter as tk
from tkinter import ttk

Why? We import the first line by tkinter as tk to shorten its name. Second, we import ttk to make our app more professional. This is, on the one hand, for readability, this is very important in Python. On the other hand, if we use a wildcard import with ttk, which is quite common: 

from tkinter import *
from tkinter.ttk import *

then the ttk widgets simply overwrite the tkinter widgets, which is sometimes not appropriate.

Notes

This site assumes that you're somewhat familiar with Python and that you're using Python 3, not Python 2. There are some significant differences between the two, including module naming, which is the first thing you'll encounter when trying tkinter.

We do not use plain tkinter widgets in our tutorials, only if it is specifically needed, and I recommend this to anyone who uses tkinter.

At this site we use tkinter as tk, and ttk as ttk you probably already know why.

Tkinter vs PyQt

Basis Tkinter PyQt
License Tk is available under BSD license, it's very liberal, and since tkinter is built in the standard Python library, it's free for any commercial use. PyQt is available under Riverbank Commercial License and GPLv3. If you do not wish to release your application under a GPL-compatible license, you must apply for a commercial license.
Design Unfortunately tkinter has a really ugly appearance, but if you use ttk perhaps with ttkthemes, you can make it modern too! PyQt has a quite nice look, and you can easily style your widgets.
Widgets Tkinter doesn't have many widgets by default (40, as I know), but you can create your own, or here's a nice package of several fantastic and useful widgets: ttkwidgets. PyQt has almost every imaginable widget with lots of useful features.
Ease of understanding Tkinter is easy to understand and master. It doesn't have much complicated part and you don't need to write object-oriented code. PyQt requires a lot of time for understanding all the details. It's object oriented, so if you donâ€™t know it, you have to learn it too.
GUI builder Tkinter doesn't have any official GUI builder, but there's a nice GUI builder (made in tkinter): Pygubu. PyQt have a Qt Designer tool which we can use to build GUIs and then get the Python code.
Installation It is included in the standard Python library, so no need to install it separately. You have to install PyQt separately, usually with pip.
