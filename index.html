What is Tkinter?

Well, if you’re on this site, you probably know what it is, but you can read about it in detail here.

Tkinter is Python's built-in standard GUI framework, which is a wrapper around the Tk GUI toolkit. Tk is written in the Tcl language but also used in other languages (Python, Ruby, Perl) and obviously the Tk interface is the module that gives us access to that. Tkinter is included with standard Windows, Linux and Mac installs of Python, so you don't need to install it separately, and it's cross-platform. Tkinter is really easy to learn really easy to use, but it looks terrible by default.

A short introduction

Most of the time, the tkinter and the ttk modules are needed when using tkinter, but there are some accessories, such as the filedialog and the messagebox.

The tk interface itself is located in a binary module called _tkinter. This module contains the low-level interface to tk, and should never be used directly by application programmers. It is usually a shared library, or DLL, but might in some cases be statically linked with the Python interpreter.

There is a module, tkinter.constants, where all tkinter constants are defined that can be used for keywords, but of course we can also use strings instead. If you import tkinter, this module will be automatically imported, so you don't have to do it separately.

Hello, ttk!

If you’ve ever created a tkinter app, you know it looks awful. Its appearance could be compared with Win95. It was not today, mainly not in design. Therefore, tkinter (ie tk) has an extension module, called ttk. With that, you can make quite modern (platform-dependent) looking apps. This appearance is native to the operating system, so it looks different from Windows and Mac. Unfortunately, on Linux, there's not a big difference between the plain tkinter and the ttk widgets.

Another good thing that with ttk you able to set different themes (hence its name: themed tk). So if you’re not happy with the look of your app, you can create your own theme, or use great libraries like ttkthemes or ttkbootstrap, for theming it easily.
Last but not least, ttk also has some new widgets compared to tkinter, but others are missing from it, so pay attention to which module you are using for the widget.

Importing

And then now a few words about importing, as many people don’t use it completely correctly. Most tkinter apps begin like this: 

from tkinter import *

this is enough for a hello world program, but not good at other times, especially if you use ttk. We recommend this import form: 

import tkinter as tk
from tkinter import ttk

Ok, but why? At the first line we import tkinter as tk, to shorten its name, but keep the namespace for better readability. import the first line by tkinter as tk to shorten its name. Second, we import ttk to make our app more professional.

Notes for using this site

This site assumes that you're somewhat familiar with Python and that you're using Python 3, not Python 2. There are some significant differences between the two, including module naming, which is the first thing you'll encounter when trying tkinter.

We don't use plain tkinter widgets in our tutorials, only if it's specifically needed, and I recommend this to anyone who uses tkinter.

At this site, we use tkinter as tk, and ttk as ttk you probably already know why.
