tk.Tk

Tk is basically the main window of tkinter. Creating a tk.Tk instance initializes the tcl/tk interpreter and creates the root window. If you don't explicitly create it, one will be implicitly created when you create your first widget. It will be stored in a global variable called _default_root. If a widget doesn't have a parent specified than it will automatically have this instance as its parent.

Methods

.aspect(minNumer, minDenom, maxNumer, maxDenom)

If all the values are specified, then they will be passed to the window manager and the window manager should use them to enforce a range of acceptable aspect ratios for window.

The aspect ratio of window (width/length) will be constrained to lie between minNumer/minDenom and maxNumer/maxDenom.

If all the values are empty strings then any existing aspect ratio restrictions are removed.

Same as wm_aspect

.attributes(option, value, option, value, ...)

Sets the window attributes. The syntax is: one of the following values, (it's important that the hyphen is not left out), then the value can be specified as a separate attribute. You can pass any number of key-value pairs to the function, but the order is important.

-alpha float
Controls the window opacity from 0.0(transparent) to 1.0(opaque). Unfortunately, this not only controls the transparency of the window content, but also the titlebar and window border, so the application can look quite weird. May not work on many Linux systems.
-disabled bool
Doesn't allow to interact with the window, if you try, you'll get a notification sound. Removes the focus.
-fullscreen bool
Controls whether the window fills the whole screen, including taskbar. If don't create possibility to close the window, you may have to use Alt + F4 or Cmd + W on Mac.
-modified bool
Controls whether the window is marked as being modified (different close button).
-notify bool
Controls whether the window's icon bounces in the dock.
-titlepath filename
Associates a (draggable) file icon with the window title that can be used to manage the given file with other applications.
-toolwindow bool
Makes a window with a single close-button (smaller than usual) on the right of the title bar, just like with torn off menus.
-topmost bool
Makes sure the window always stays on top of all other windows, even if it has no focus.
-transparent or -transparentcolor color
Makes all areas of the window transparent and cursor penetrable with the specified color. Unfortunately you can't control this transparency.
-type list

Sets the type of top-level window. The most of this options is unnecessary, but splash and dialog can be useful.

Options:

  • combo
  • dnd
  • desktop
  • dialog
  • dock
  • dropdown_menu
  • menu
  • normal
  • notification
  • popup_menu
  • splash
  • tooltip
  • toolbar
  • utility
-zoomed bool
Controls whether the window is in zoomed state (because .state('zoomed') doesn't work on X11)

Same as wm_attributes

.deiconify()

Deiconify the window.

Same as wm_deiconify

.geometry(newGeometry)

The function takes a single parameter as attribute, the geometry must be specified as a single string, yep, this solution is kinda weird. The syntax is: "widthxheight+x+y" as one single string.

If you want to specify just the width and height then pass "300x300" to the argument, of course with your own values.

If you want to specify just the x and y position then pass "+300+300" to the argument, of course with your own values. It's important that the + sign is not left out from the beginning of the string.

And of course you can define all the 4 values the same time. "300x300+300+300"

Same as wm_geometry

.iconify()

Iconify the window, so it'll appear as an icon on the taskbar/dock.

Same as wm_iconify

.title(string)

Sets the window title. This appears in the title bar of the window and on the taskbar/dock icon or its tooltip. The only attribute is the string, which of course means the title string.

Same as wm_title
